# OpenClaw Web Manager (OWM) - Project Memory

## 핵심 비전: Star-Office-UI 게이미피케이션

OWM의 핵심 차별점은 **게임화된 에이전트 관리 UI**:

### 롤모델
- Star-Office-UI (픽셀아트 가상 오피스)
- Pixel Agents, Agents in the Office 등 2025 트렌드

### 3가지 게임 메타포
1. **모니터링 = 게임 관전**: 에이전트 캐릭터가 상태에 따라 환경에서 일하거나 쉬거나 (idle/working/speaking/tool_calling/error)
2. **설정 = 스탯/장비**: 에이전트 설정을 RPG 스탯처럼 보고 변경 (포트→공격력, 로그레벨→시야, 모델→직업)
3. **스킬/커맨드/시스템프롬프트 = 아이템**: AI 표준 요소를 장비처럼 추가/제거/교체

### 목표
- CLI로만 가능하던 에이전트 관리를 게임처럼 친숙한 GUI로
- 기술자가 아닌 사람도 에이전트를 직관적으로 관리
- 실시간 WebSocket 기반 상태 시각화

## 자기 검증 규칙

- 주장하기 전에 반드시 검증할 것. 검증 수단:
  - 코드베이스 주장 → Grep/Read로 실제 파일/API 존재 확인
  - 빌드/타입 주장 → `npm run build` 또는 `npx tsc --noEmit`으로 증명
  - 외부 라이브러리 주장 → WebSearch로 공식 문서 확인
  - Claude Code 기능 주장 → WebSearch 또는 직접 테스트로 확인
- 검증 없이 "~일 것이다", "~로 알고 있다"로 답변하지 말 것
- 확인 못 하면 "미확인"이라고 밝힐 것

## 자율 판단 규칙

- **구현 판단은 유저에게 묻지 않는다** — 어떤 접근법을 쓸지, 어떤 순서로 할지는 스스로 결정
- 판단이 필요하면 조사(WebSearch, 코드베이스 탐색)하거나 에이전트끼리 토론으로 해결
- 유저에게 묻는 경우: 비즈니스 요구사항이 불명확할 때만 (기술 구현 방법은 묻지 않음)
- AskUserQuestion 사용 최소화 — 유저는 결과를 원하지 과정 확인을 원하지 않는다

## 오케스트레이터 검증 (스프린트 파이프라인)

> 게이트 상세 정의는 `docs/agents/pipeline.md`의 3계층 검증 아키텍처 참조.
> 아래는 오케스트레이터(PM)의 **행동 규칙**만 기술한다.

서브에이전트 산출물을 받은 후, 다음 Phase로 넘기기 전에 **PM이 직접 도구를 실행**하여 검증한다.
에이전트에게 "확인하세요"라고 지시하는 것이 아닌, PM이 직접 결과를 본다.

- **Phase 2 후**: 02-review.md의 각 지적에 `[근거:]` 태그가 있는지 확인. 근거 없는 심각도 "높음" 지적이 있으면 Grep/WebSearch로 직접 확인 후, 틀렸으면 제거하고 Phase 3에 전달.
- **Phase 4 후**: 04-implementation.md의 변경 파일 목록과 03-spec-revised.md를 대조. 스펙에 언급되지 않은 파일 수정이 있으면:
  - 공유 타입/유틸 파일 → 허용 (스펙 구현에 필수적인 연쇄 수정)
  - `components/ui/` 수정 → 경고 (기존 컴포넌트 변경은 회귀 위험, 근거 확인)
  - 스펙과 무관한 파일 → 위반 (FE_DEV에게 되돌리기 요청)
- **Phase 5 후**: 05-qa-report.md의 FAIL 항목에 측정값과 **근본 원인 분석**이 있는지 확인. `[측정불가]`만으로 FAIL 판정한 항목은 직접 재검증. 근본 원인 없는 FAIL은 QA에게 분석 보완 요청.
- 검증 결과 수정이 필요하면, 해당 산출물을 인라인 수정하고 수정 사유를 주석으로 남긴다.

## 스프린트 상태 관리

- 각 스프린트 폴더에 `.state.json`으로 진행 상태를 추적한다
- Phase 완료마다 상태 파일을 업데이트한다
- 오토컴팩트 후에도 `.state.json`을 읽어 복구할 수 있다
- `/sprint resume`로 중단된 스프린트를 재개할 수 있다
- Layer 1 Stop 훅이 게이트 결과를 `.state.json`의 `gates` 객체에 자동 기록한다

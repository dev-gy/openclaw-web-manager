# Sprint-002: 00-plan.md -- 유저 분석 + 기획

## 1. 유저 분석

### 1.1 유저는 누구인가

| 항목 | 설명 |
|------|------|
| 역할 | AI 에이전트 운영자 (개발자 또는 기술 관리자) |
| 숙련도 | CLI에 익숙하지만, 대시보드 UI를 선호하는 유저. 게임 UI에 친숙한 20-40대 |
| 주요 목표 | 현재 동작 중인 에이전트들의 상태를 한눈에 파악하고, 이상 징후를 빠르게 감지 |
| 2차 목표 | 에이전트 개별 상세 정보 확인 (얼마나 일했는지, 어떤 플랫폼인지, 마지막 활동) |
| 사용 맥락 | 데스크톱 브라우저 (1024px 이상), 업무 중 한쪽 모니터에 상시 띄워두는 용도 |

### 1.2 현재 유저 경험 (실제 코드 확인 결과)

**진입 경로**: 사이드바 "에이전트 오피스" 클릭 -> `/monitor/agents`

**현재 화면 구조** (`pages/monitor/agents/+Page.tsx` 확인):

1. **헤더**: "에이전트 오피스" 제목 + 실시간/활동중/총원 배지 + 새로고침 버튼
2. **오피스 레이아웃**: `grid grid-cols-1 lg:grid-cols-3` 그리드에 5개 OfficeZone 배치
   - 작업 데스크 (lg:col-span-2), 미팅 룸, 도구 창고, 라운지, 입구
3. **각 OfficeZone**: `rounded-xl border p-4 min-h-[140px]` + 그라데이션 배경 직사각형
   - 에이전트 있으면: `flex flex-wrap gap-3`으로 AgentCharacter 나열
   - 에이전트 없으면: `<span className="text-xs italic">비어있음</span>`
4. **AgentCharacter**: AgentAvatar(SVG 캐릭터) + 이름(10px) + 플랫폼(9px) 세로 배치
5. **AgentDetailPanel**: 선택 시 하단에 Card로 RPG 스탯 시트 표시

**유저가 겪는 불편함/혼란 5가지**:

| # | 불편함 | 근거 (코드 확인) |
|---|--------|-----------------|
| P1 | **"이게 게임 오피스?"**: 색상 그라데이션 직사각형 5개가 나열되어 있을 뿐, 가구/바닥/소품이 없어 "대시보드 카드 5개"로 보임 | `OfficeZone`은 `div.rounded-xl.border`에 텍스트 제목만 가진 구조. 배경 SVG/가구 SVG 전무 |
| P2 | **빈 상태가 텍스트 1줄**: 에이전트가 없는 존은 `text-xs italic "비어있음"` 14px 이탤릭 한 줄. 빈 오피스도 "게임 환경"으로 보여야 하는데, 텍스트 하나로는 공간감이 없음 | `OfficeZone` line 228-230: `<span className="text-xs italic">비어있음</span>` |
| P3 | **에이전트가 환경 "위에" 있지 않음**: AgentAvatar가 flexbox gap으로 나열될 뿐, 오피스 바닥 위에 서있는 공간적 배치가 없음. 캐릭터가 카드 안에 갇혀 있음 | `AgentCharacter`는 `button.flex.flex-col.items-center` 안에 AgentAvatar + 텍스트 |
| P4 | **에이전트가 지금 뭘 하고 있는지 모름**: `lastAction` 텍스트가 `title` 속성에만 있어 호버 해야 보임. 말풍선이나 활동 로그 없음 | `AgentCharacter` line 255: `title={\`${agent.name} -- ${agent.lastAction}\`}` |
| P5 | **전체 오피스 조감 불가**: 5개 존이 grid 카드로 나뉘어 각각 독립된 영역처럼 보임. "한 공간 안에 여러 구역"이 아닌 "5개 별도 카드" | grid layout + 각 존이 개별 border/bg를 가짐 |

### 1.3 유저가 진짜 원하는 것

| 표면적 요구 | 근본적 필요 |
|------------|-----------|
| "오피스가 게임처럼 보였으면" | **자기 에이전트들이 실제로 "일하고 있다"는 감각적 확신** -- 숫자/텍스트가 아닌 시각적 움직임으로 상태 인지 |
| "빈 오피스가 허전하지 않았으면" | **에이전트가 없어도 "여기에 올 수 있다"는 기대감** -- 빈 책상, 빈 의자가 있는 준비된 공간 |
| "에이전트가 뭘 하고 있는지 바로 보고 싶다" | **호버/클릭 없이 현재 상태를 1초 이내에 파악** -- 말풍선, 상태 아이콘, 위치 자체가 정보 |
| "하나의 오피스 공간처럼 보였으면" | **5개 존이 하나의 연속된 공간** -- 벽/바닥이 이어지는 등각 투영(isometric) 또는 2D 사이드뷰 |

## 2. 아이디어 도출

### 아이디어 A: 등각 투영(Isometric) 2.5D 오피스 (미채택)

- **해결하는 문제**: P1(게임 느낌 없음), P5(조감 불가)
- **설명**: 45도 기울어진 등각 투영 뷰로 오피스 전체를 한 화면에 표시. 에이전트가 타일 위를 걸어다님
- **미채택 이유**: CSS/SVG만으로 등각 좌표 변환, 타일맵 렌더링, z-ordering을 구현하면 복잡도가 과도. 캔버스(Canvas) 또는 PixiJS 같은 렌더링 엔진이 필요하며, 기존 React/Tailwind 스택과 이질적. Sprint 1개로 완성 불가능한 규모

### 아이디어 B: 2D 사이드뷰 싱글 플로어 오피스 (채택)

- **해결하는 문제**: P1, P2, P3, P4, P5 전부
- **설명**: 하나의 넓은 가로 패널 안에 바닥선/벽면/천장이 있고, 그 위에 5개 구역을 좌→우로 배치. 각 구역에 가구(SVG)가 고정 배치되어 있고, 에이전트 캐릭터가 해당 구역의 바닥 위에 서있음. 빈 구역도 가구는 그대로 보임
- **채택 이유**:
  - SVG + CSS positioning으로 구현 가능 (외부 의존성 없음)
  - 기존 `AgentAvatar` 컴포넌트를 그대로 재사용
  - 빈 오피스에도 가구가 보여 "준비된 공간" 느낌
  - 가로 스크롤 또는 반응형 축소로 전체 조감 가능
  - 에이전트와 환경이 같은 좌표 공간에 있어 "위에 서있는" 느낌
  - "한 공간의 여러 구역"으로 P5 해결

### 아이디어 C: 말풍선 + 상태 인디케이터 (채택)

- **해결하는 문제**: P4 (에이전트 활동 불투명)
- **설명**: 에이전트 캐릭터 위에 말풍선이 떠서 `lastAction` 텍스트를 표시. 상태별 이모티콘 아이콘이 말풍선 안에 포함. 말풍선은 3초 표시 후 페이드아웃, 새 활동이 오면 다시 표시
- **채택 이유**: 호버 없이 즉시 상태 파악 가능. 게임에서 NPC 머리 위 말풍선과 동일한 패턴

### 아이디어 D: 오피스 하단 활동 로그 피드 (채택)

- **해결하는 문제**: P4 (활동 불투명), 시간순 이력 필요
- **설명**: 오피스 뷰 하단에 최근 10건의 에이전트 활동을 시간순으로 표시하는 터미널 스타일 피드. "[시간] 에이전트명: 활동내용" 형식. WebSocket으로 실시간 업데이트
- **채택 이유**: 말풍선은 현재 상태만 보여주지만, 로그 피드는 시간순 이력 제공. 기존 Terminal 컴포넌트 스타일 재활용 가능

### 아이디어 E: 존별 가구/소품 SVG 세트 (채택)

- **해결하는 문제**: P1(게임 느낌 없음), P2(빈 상태)
- **설명**: 각 존에 고유 가구를 SVG로 배치
  - desk: 모니터가 있는 책상 2개, 의자 2개
  - meeting: 둥근 테이블 1개, 의자 3개, 화이트보드 1개
  - bookshelf: 책장 2개, 사다리 1개
  - lounge: 소파 1개, 커피 테이블 1개, 화분 1개
  - entrance: 문 1개, 우산꽂이 1개, 신발장 1개
- **채택 이유**: 빈 존에도 가구가 보여 "오피스 공간" 느낌 유지. SVG로 구현하여 외부 이미지 불필요. 다크/라이트 테마 CSS 변수로 색상 전환

### 아이디어 F: 미니맵 네비게이션 (미채택)

- **해결하는 문제**: P5 (전체 조감)
- **설명**: 오피스 뷰 오른쪽 상단에 축소된 전체 레이아웃 미니맵 표시. 클릭으로 해당 구역으로 스크롤
- **미채택 이유**: 아이디어 B(싱글 플로어)로 전체가 한 화면에 보이면 미니맵 불필요. 추후 오피스 크기가 커지면 재고

### 아이디어 G: 오피스 배경 파티클 효과 (미채택)

- **해결하는 문제**: P1 (게임 느낌)
- **설명**: 배경에 먼지, 반짝이, 커피 김 같은 파티클 효과
- **미채택 이유**: 성능 부담 대비 정보 전달 가치 없음. 모니터링 도구의 핵심 목적(상태 파악)과 무관한 장식. 추후 Easter egg로 고려

## 3. 목표 정의

### 3.1 유저 가치 (이 스프린트로 달성할 것)

1. **"게임 관전 화면" 느낌**: `/monitor/agents` 진입 시 유저가 "대시보드"가 아닌 "오피스"를 보고 있다고 느낀다
2. **즉시 상태 파악**: 에이전트의 현재 상태를 호버/클릭 없이 1초 이내에 시각적으로 인지한다 (위치 + 말풍선 + 애니메이션)
3. **빈 오피스도 공간**: 에이전트가 0명이어도 가구가 배치된 오피스가 보여, "에이전트가 오면 여기서 일할 것"이라는 기대감이 있다
4. **하나의 연속 공간**: 5개 존이 하나의 바닥/벽면 위에 연속적으로 배치되어 "한 오피스의 여러 구역"으로 인지된다

### 3.2 성공 지표

| 지표 | 측정 방법 | 기준 |
|------|----------|------|
| 오피스 배경 SVG 존재 | 코드에 바닥선/벽면/가구 SVG 요소가 렌더링됨 | 5개 존 모두 가구 SVG 포함 |
| 빈 상태에서 가구 표시 | agents=[] 일 때도 가구 SVG가 화면에 표시됨 | "비어있음" 텍스트가 제거되고 가구만 보임 |
| 말풍선 표시 | working/speaking/tool_calling 상태 에이전트에 말풍선 요소 존재 | 말풍선이 캐릭터 위에 렌더링됨 |
| 활동 로그 피드 | 오피스 하단에 최근 활동 목록 표시 | 최소 1건 이상의 로그 항목이 렌더링됨 |
| 싱글 플로어 레이아웃 | 5개 존이 하나의 컨테이너 안에 가로 배치 | grid 5개 카드가 아닌 1개 continuous container |
| npm run build 통과 | 빌드 실행 | exit code 0 |
| 기존 인터페이스 유지 | useAgentActivity 훅의 반환값, AgentAvatar props 변경 없음 | 타입 호환성 유지 |

### 3.3 범위 제한 (하지 않는 것)

- 백엔드 API 변경 (useAgentActivity 훅의 데이터 소스 그대로)
- AgentAvatar의 SVG 캐릭터 디자인 변경 (기존 레트로 스타일 유지)
- AgentDetailPanel의 RPG 스탯 시트 구조 변경 (내용 유지, 위치만 조정 가능)
- 등각 투영/3D 변환
- 에이전트 드래그앤드롭 위치 변경
- 사이드바 메뉴 구조 변경 (별도 스프린트)

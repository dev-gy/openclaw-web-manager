## Section A: 기능 요구사항

### F-001: 싱글 플로어 오피스 컨테이너

**유저 문제 해결**: 5개 grid 카드로 분리된 존을 하나의 연속 공간으로 통합하여, "한 오피스의 여러 구역"으로 인지하게 한다.

**상세**:
- 기존 `grid grid-cols-1 lg:grid-cols-3 gap-4`를 제거하고, 1개의 가로 컨테이너로 대체
- 컨테이너 내부에 바닥선(floor line), 벽면(wall), 천장 라인이 SVG로 그려진다
- 바닥선은 컨테이너 하단 20% 위치에 수평선으로 표시
- 벽면은 컨테이너 상단 80% 영역을 차지하며, 벽 색상은 CSS 변수 `--owm-bg-secondary` 사용
- 5개 존은 바닥선 위에 좌→우로 순서대로 배치: entrance(입구) | lounge(라운지) | desk(작업) | meeting(미팅) | bookshelf(도구)
- 각 존 사이에 점선 구분선(높이 60%, 상단 정렬)을 표시하여 구역 경계를 암시
- 컨테이너 최소 높이: 420px, 최대 높이: 520px
- 가로 폭: 부모 컨테이너 100% (min-width: 800px, 이하에서는 가로 스크롤)
- 각 존의 가로 폭 비율: entrance 12% | lounge 18% | desk 32% | meeting 20% | bookshelf 18%

**입출력**:
- 입력: `agents: AgentActivity[]` (useAgentActivity 훅에서)
- 출력: SVG + HTML 하이브리드 레이아웃 (배경은 SVG, 에이전트/가구는 absolute positioned HTML)

**제약 조건**:
- CSS 변수 `--owm-bg-*`, `--owm-border` 사용하여 다크/라이트 모드 호환
- 1024px 미만 화면에서는 `overflow-x: auto` 가로 스크롤 허용

### F-002: 존별 가구 SVG 세트

**유저 문제 해결**: 빈 존에도 가구가 보여 "게임 오피스" 느낌을 유지하고, 에이전트 없어도 공간감을 제공한다.

**상세**: 각 존에 아래 가구를 SVG 인라인으로 렌더링한다. 가구는 바닥선 위에 고정 배치되며, 에이전트 유무와 무관하게 항상 표시된다.

| 존 | 가구 목록 | 비고 |
|-----|----------|------|
| entrance | 문(열린 상태) 1개, 우산꽂이 1개 | 문은 존의 왼쪽 끝, 우산꽂이는 문 옆 |
| lounge | 소파 1개, 커피 테이블 1개, 화분 1개 | 소파는 벽 쪽(상단), 테이블은 소파 앞 |
| desk | 모니터 달린 책상 2개, 사무용 의자 2개 | 좌우로 2세트, 각 책상에 모니터 1개 |
| meeting | 둥근 테이블 1개, 의자 3개, 화이트보드 1개 | 화이트보드는 벽 쪽(상단) |
| bookshelf | 책장 2개, 스텝 사다리 1개 | 책장은 벽 쪽(상단), 사다리는 앞에 |

**가구 SVG 스타일**:
- 선 두께(stroke-width): 1.5px
- 선 색상: `var(--owm-text-secondary)` (opacity 0.5)
- 채움 색상: `var(--owm-bg-card)` (opacity 0.3)
- 전체적으로 단순한 2D 사이드뷰 스타일 (레트로 게임 배경과 일치)
- 가구 높이: 바닥선으로부터 40px~80px (가구 유형별 차이)

**제약 조건**:
- 외부 이미지 파일 사용 금지 (모두 인라인 SVG)
- 가구는 interactive하지 않음 (클릭 이벤트 없음, pointer-events: none)

### F-003: 에이전트 캐릭터 공간 배치

**유저 문제 해결**: 에이전트가 카드 안에 갇혀있지 않고, 오피스 바닥 위에 서있는 느낌을 준다.

**상세**:
- 에이전트 캐릭터는 해당 존의 바닥선 위에 `position: absolute`로 배치
- 바닥선의 y 좌표를 기준으로, 캐릭터의 bottom이 바닥선에 닿도록 위치
- 같은 존에 복수 에이전트가 있으면, 존 내에서 좌→우로 40px 간격으로 배치
- 에이전트 최대 표시 수: 존당 6명 (7명 이상이면 "+N" 뱃지 표시)
- 에이전트 아바타 크기: 오피스 뷰에서는 `size="md"` (48px) 사용
- 캐릭터 아래에 이름 라벨 표시 (font-size: 10px, max-width: 52px, truncate)
- 클릭 시 `selectedAgent` 설정 (기존 동작 유지)
- 선택된 에이전트: 캐릭터 아래에 삼각형 화살표(highlight indicator) 표시

**에이전트 위치 계산**:
- 존의 시작 x좌표 + 가구 배치 이후의 빈 공간에 에이전트 배치
- desk 존: 책상 뒤(상단)에 의자에 "앉아있는" 느낌으로 배치 (y좌표를 바닥선보다 약간 위로)
- lounge 존: 소파 앞 바닥에 서있거나 앉아있는 위치
- meeting 존: 테이블 주변에 둘러앉은 위치
- entrance 존: 문 근처 바닥에 서있는 위치
- bookshelf 존: 책장 앞 바닥에 서있는 위치

### F-004: 에이전트 말풍선

**유저 문제 해결**: 호버/클릭 없이 에이전트의 현재 활동을 1초 이내에 파악한다.

**상세**:
- 에이전트 캐릭터 상단에 말풍선 UI 표시
- 말풍선 내용: 상태 아이콘 + lastAction 텍스트
  - working: "작업 중..." (키보드 아이콘)
  - speaking: "대화 중" (말풍선 아이콘)
  - tool_calling: "도구 실행 중" (렌치 아이콘)
  - idle: 말풍선 표시하지 않음 (Zzz 이모티콘만 캐릭터 위에 떠있음)
  - error: "오류 발생!" (경고 아이콘, 빨간색 배경)
  - offline: 말풍선 표시하지 않음
- 말풍선 스타일:
  - 배경: `var(--owm-bg-card)`, 테두리: `var(--owm-border)`, 모서리 반경: 8px
  - error 상태 시 배경: `var(--owm-error)` opacity 0.1, 테두리: `var(--owm-error)` opacity 0.3
  - 최대 너비: 120px, 텍스트 truncate
  - 하단에 삼각형 꼬리(tail) 표시
  - 폰트 크기: 10px
- 말풍선 애니메이션:
  - 나타날 때: opacity 0->1 + translateY(-4px->0) (200ms ease-out)
  - 사라질 때: opacity 1->0 (300ms ease-in)
- 말풍선 표시 타이밍:
  - 상태 변경 시 말풍선 표시
  - working/speaking/tool_calling: 상시 표시 (사라지지 않음)
  - error: 상시 표시
  - idle: Zzz 부유 애니메이션만 (기존 agent-idle 애니메이션 활용)

### F-005: 존 라벨 및 구역 표시

**유저 문제 해결**: 유저가 각 구역의 의미(어떤 상태의 에이전트가 여기 있는지)를 즉시 파악한다.

**상세**:
- 각 존의 상단(벽면 영역)에 존 이름 라벨을 표시
- 라벨은 "간판" 스타일로 디자인 (직사각형 배경 + 텍스트)
- 라벨 내용:
  - entrance: "입구" (아이콘: 문)
  - lounge: "라운지" (아이콘: 커피)
  - desk: "작업 데스크" (아이콘: 모니터)
  - meeting: "미팅 룸" (아이콘: 말풍선)
  - bookshelf: "도구 창고" (아이콘: 책)
- 라벨 스타일: font-size 11px, font-weight 600, color `var(--owm-text-secondary)`, bg `var(--owm-bg-card)` opacity 0.8, padding 2px 8px, border-radius 4px
- 라벨 위치: 존 영역 상단 중앙, 벽면에 "걸려있는" 느낌 (top: 8px)
- 존에 에이전트가 1명 이상 있으면 라벨 옆에 인원수 뱃지 표시 (Badge 컴포넌트 size="sm")

### F-006: 활동 로그 피드

**유저 문제 해결**: 에이전트의 시간순 활동 이력을 실시간으로 확인한다.

**상세**:
- 오피스 뷰 하단에 가로 full-width 로그 피드 영역 표시
- 높이: 120px (고정), overflow-y: auto (스크롤 가능)
- 배경: `var(--owm-bg-card)`, border-top: `var(--owm-border)`
- 각 로그 항목 형식: `[HH:MM:SS] 에이전트명: 활동내용`
- 로그 폰트: font-mono (JetBrains Mono), font-size: 11px
- 로그 색상: 시간은 text-secondary, 에이전트명은 에이전트의 avatarSeed 기반 색상, 활동내용은 text-primary
- 로그 소스: useAgentActivity의 WebSocket onMessage에서 session.started / session.updated / session.ended 이벤트를 캡처
- 최대 50건 유지 (FIFO -- 51번째 항목이 오면 1번째 삭제)
- 빈 상태: "아직 활동 기록이 없습니다. 에이전트가 활동하면 여기에 표시됩니다." (text-secondary, italic, 중앙 정렬)
- 새 로그 추가 시 자동 스크롤 (scrollTop = scrollHeight)
- 로그 피드 접기/펼치기 토글 버튼 (ChevronDown/ChevronUp 아이콘, 우상단)
  - 접힌 상태: 높이 32px, 마지막 1건만 표시
  - 펼친 상태: 높이 120px (기본)

### F-007: 오피스 헤더 개선

**유저 문제 해결**: 오피스 상태를 헤더에서 빠르게 확인하되, 게임 HUD 느낌을 유지한다.

**상세**:
- 기존 헤더 구조를 유지하되, 디자인 미세 조정:
  - 제목: "에이전트 오피스" (이모지 유지)
  - 실시간 배지: wsConnected 상태에 따라 녹색/빨간색 점 + "실시간"/"연결 끊김" 텍스트
  - 활동중/총원 배지 유지
  - 새로고침 버튼 유지
- 추가 요소: 현재 시간 표시 (HH:MM 형식, 1분 간격 갱신) -- 오피스 시계 느낌
- 시간 표시 스타일: font-mono, font-size 14px, text-secondary

### F-008: 로딩 상태 오피스 스켈레톤

**유저 문제 해결**: 데이터 로딩 중에도 "오피스"가 로드되고 있음을 시각적으로 전달한다.

**상세**:
- 기존 스피너 + "오피스를 불러오는 중..." 대신, 오피스 실루엣 스켈레톤 표시
- 스켈레톤 구성:
  - 오피스 컨테이너와 동일한 크기(높이 420px, 가로 100%)
  - 바닥선 위치에 가로 선 (animate-pulse)
  - 5개 존 위치에 직사각형 블록 (animate-pulse, 각각 약간 다른 크기)
  - 가구 위치에 작은 직사각형 (animate-pulse, opacity 0.3)
- 애니메이션: Tailwind `animate-pulse` 사용 (기존 LoadingSkeleton 패턴과 일치)

### F-009: 에이전트 0명 빈 오피스

**유저 문제 해결**: 에이전트가 없어도 "준비된 공간"으로 보여, 연결 후 에이전트가 올 것이라는 기대감을 준다.

**상세**:
- agents 배열이 빈 경우:
  - 오피스 배경(바닥/벽/가구)는 정상 표시
  - 기존 Card + "오피스가 비어있습니다" 텍스트를 제거
  - 오피스 중앙(desk 존 위치)에 반투명 안내 표시:
    - 텍스트: "에이전트를 기다리는 중..."
    - 부제: "에이전트 세션이 시작되면 오피스에 자동으로 나타납니다"
    - 폰트: 텍스트 14px semibold, 부제 12px regular
    - 색상: text-secondary (opacity 0.6)
    - 반투명 배경 패널: bg-card/60, backdrop-blur-sm, rounded-lg, padding 16px 24px
  - 활동 로그 피드는 빈 상태 메시지 표시

### F-010: AgentDetailPanel 위치 변경

**유저 문제 해결**: 에이전트 상세 정보를 오피스 뷰 옆에서 확인하여, 오피스 뷰를 가리지 않는다.

**상세**:
- 기존: 오피스 뷰 하단에 Card로 표시 (오피스 뷰를 밀어냄)
- 변경: 오피스 오른쪽에 사이드 패널(슬라이드 인)로 표시
- 사이드 패널 스타일:
  - width: 320px (고정)
  - 높이: 오피스 컨테이너와 동일
  - 배경: var(--owm-bg-card), border-left: var(--owm-border)
  - 등장 애니메이션: translateX(100%) -> translateX(0) (200ms ease-out)
  - 닫기 버튼: 우상단 X 아이콘
- 패널이 열리면 오피스 컨테이너 가로 폭이 `calc(100% - 320px)`로 줄어듦
- 패널 내용: 기존 AgentDetailPanel 내용 그대로 유지 (아바타, 이름, RPG 스탯, 경험치 바)
- 1280px 미만 화면에서는 사이드 패널 대신 오버레이 모달로 표시 (z-index: 50, backdrop)

## Section D: 수용 기준 (Acceptance Criteria)

### AC-001: 유저가 오피스 전체를 하나의 공간으로 볼 수 있다

- Step 1: `/monitor/agents` 페이지 진입 (Gateway 연결 상태)
- Step 2: 오피스 영역 확인
- 기대 결과: 5개 존이 하나의 연속된 컨테이너 안에 가로로 배치되어 있다. 별도의 border/gap으로 분리된 카드 5개가 아니라, 바닥선이 이어지는 하나의 공간이다
- 실패 조건: 존별로 독립된 카드/border가 보임. 또는 grid gap으로 존 사이에 빈 공간이 있음

### AC-002: 유저가 빈 오피스에서 가구를 볼 수 있다

- Step 1: `/monitor/agents` 진입 (에이전트 0명 상태)
- Step 2: 오피스 영역 확인
- 기대 결과: 각 존에 가구(문, 소파, 책상, 화이트보드, 책장)가 SVG로 표시됨. "비어있음" 텍스트가 아닌 실제 가구 그래픽이 보임. 중앙에 "에이전트를 기다리는 중..." 안내 패널이 보임
- 실패 조건: 가구 없이 빈 영역만 보임. "비어있음" 텍스트만 표시됨. 오피스 배경 자체가 보이지 않음

### AC-003: 유저가 에이전트의 상태를 호버 없이 파악할 수 있다

- Step 1: `/monitor/agents` 진입 (에이전트 1명 이상 working 상태)
- Step 2: 에이전트 캐릭터 확인 (호버하지 않음)
- 기대 결과: working 에이전트가 desk 존에 위치하며, 캐릭터 위에 "작업 중..." 말풍선이 표시되고, working 애니메이션이 동작함. 캐릭터 아래에 이름이 보임
- 실패 조건: 말풍선 없음. 상태를 파악하려면 호버/클릭 필요. 에이전트가 어느 존에 있는지 구분 안 됨

### AC-004: 유저가 에이전트를 클릭하여 상세 정보를 볼 수 있다

- Step 1: `/monitor/agents`에서 에이전트 캐릭터 클릭
- Step 2: 사이드 패널 확인
- 기대 결과: 오른쪽에 사이드 패널이 슬라이드되어 나타남. 패널에 에이전트의 아바타(lg), 이름, 상태 뱃지, 플랫폼, 메시지 수(EXP), 가동 시간, 마지막 활동, 경험치 바가 표시됨
- 실패 조건: 패널 미표시. 오피스 뷰 하단에 카드로 표시됨 (기존 방식). 패널에 에이전트 정보 미표시

### AC-005: 유저가 활동 로그를 확인할 수 있다

- Step 1: `/monitor/agents` 진입 (에이전트 1명 이상)
- Step 2: 오피스 하단의 활동 로그 영역 확인
- 기대 결과: "[HH:MM:SS] 에이전트명: 활동내용" 형식의 로그가 시간순으로 나열됨. 새 이벤트 발생 시 자동으로 추가됨
- 실패 조건: 로그 영역 미표시. 로그 항목 없음 (에이전트 존재에도). 자동 업데이트 안 됨

### AC-006: 유저가 로딩 중 오피스 스켈레톤을 볼 수 있다

- Step 1: `/monitor/agents` 진입 시 데이터 로딩 중 확인
- 기대 결과: 오피스 크기의 스켈레톤 영역이 표시되며 animate-pulse 동작. 단순 스피너 + 텍스트가 아닌 오피스 실루엣
- 실패 조건: 기존 스피너 + "오피스를 불러오는 중..." 텍스트가 보임. 빈 화면 표시

### AC-007: 유저가 에러 시 상황을 파악하고 복구할 수 있다

- Step 1: 에러 상태 발생 (네트워크 실패 시뮬레이션)
- 기대 결과: 오피스 상단에 에러 배너 표시. 에러 메시지 + 재시도 방법이 보임. 오피스 뷰 자체는 마지막 데이터로 유지
- 실패 조건: 에러 무시. 빈 화면으로 전환. 복구 경로 없음

### AC-008: 오피스 뷰가 1024px 화면에서 깨지지 않는다

- Step 1: 브라우저 폭을 1024px로 조정
- Step 2: `/monitor/agents` 확인
- 기대 결과: 오피스 뷰가 가로 스크롤로 접근 가능하거나, 축소되어 표시됨. 레이아웃이 깨지지 않음. 존 라벨, 가구, 에이전트가 모두 보임
- 실패 조건: 요소 겹침, 잘림, 레이아웃 깨짐

### AC-009: 다크/라이트 모드 모두에서 오피스가 정상 표시된다

- Step 1: 다크 모드에서 `/monitor/agents` 확인
- Step 2: 라이트 모드로 전환 후 확인
- 기대 결과: 두 모드 모두에서 벽면, 바닥, 가구, 에이전트, 말풍선, 로그가 가독성 있게 표시됨. 색상 대비가 충분함
- 실패 조건: 특정 모드에서 요소가 안 보임 (배경과 색상 동일). 가구 SVG가 보이지 않음

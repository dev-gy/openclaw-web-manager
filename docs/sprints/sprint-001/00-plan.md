# Sprint-001: 설치 미완료 상태의 빈 화면/리다이렉트 UX 개선

## 목표

설치 마법사(setup wizard)를 완료하지 않은 유저가 대시보드, 모니터링, 설정 등 주요 페이지에 접근했을 때, **현재 상태를 즉시 이해하고 다음 행동(설치 마법사로 이동)을 3초 이내에 결정**할 수 있도록 개선한다.

## 배경

### 현재 문제

1. **guard.ts는 인증만 확인한다**: `pages/+guard.ts`는 로그인 여부(`owm_session` 쿠키)만 검사하여 미인증 시 `/login`으로 리다이렉트한다. 그러나 "로그인은 했지만 Gateway 연결(설치)은 안 된 상태"에 대한 가드가 없다.

2. **대시보드(/)의 빈 상태 처리가 불완전하다**: `pages/index/+Page.tsx`는 `useHealthData` 훅으로 연결 상태를 확인한 뒤, 미연결 시 `DisconnectedView`를 보여준다. 이 뷰는 "Gateway에 연결되지 않았습니다"라는 메시지와 `/setup/1` 링크를 제공하지만:
   - 로딩 중(`loading === true`)에는 아무 피드백도 없이 빈 화면이 나타날 수 있다.
   - `connected`와 `loading`이 모두 `false`가 될 때까지 유저는 대시보드 레이아웃(사이드바, 헤더)만 보게 된다.

3. **다른 페이지는 빈 상태 처리가 각각 다르다**:
   - `/config` (설정 인덱스): 하단에 "Gateway에 연결해야 합니다" 안내가 있지만 8개 설정 카드는 그대로 노출되어 클릭 가능 (클릭 시 빈 하위 페이지)
   - `/monitor` (모니터링 인덱스): 빈 상태 안내 없이 "연결되지 않음" 상태카드만 표시
   - `/config/agents`, `/config/channels` 등 하위 페이지: 개별적으로 연결 상태를 확인하지 않아 빈 화면 또는 에러 발생 가능

4. **유저 시나리오상 실제 발생하는 상황**:
   - 첫 설치 후 로그인했지만 설치 마법사를 완료하기 전에 사이드바 메뉴를 클릭
   - 브라우저 북마크로 대시보드 직접 접근
   - 다른 사용자가 보내준 설정 페이지 링크를 직접 오픈

### DEC-002 연관

의사결정 기록 DEC-002에서 발견된 5가지 문제 중 "빈 직사각형 5개"(에이전트 오피스), "페이지 이동 시 5-7초 로딩" 문제와 연관된다. 특히 미설치 상태에서 페이지를 돌아다니면 빈 UI가 반복 노출되어 첫인상이 크게 나빠진다.

## 유저 시나리오

### 시나리오 1: 첫 방문자 — 로그인 직후 사이드바 탐색

| 단계 | 유저 행동 | 현재 결과 | 기대 결과 |
|------|----------|----------|----------|
| 1 | 처음 OWM에 접속, 로그인 완료 | 대시보드(/) 진입 | 동일 |
| 2 | Gateway 미연결 → DisconnectedView 노출 | "Gateway에 연결되지 않았습니다" + 링크 | 동일 (이 부분은 이미 존재) |
| 3 | 호기심에 사이드바 "설정" 클릭 | 8개 설정 카드 노출 (하단에 작은 안내) | 설치 미완료 안내가 페이지 상단에 명확히 노출. 설정 카드는 비활성(disabled) 처리 |
| 4 | "캐릭터 시트" 카드 클릭 | 빈 페이지 또는 API 에러 | `/setup/1`로 리다이렉트 또는 진입 차단 안내 |

### 시나리오 2: 북마크 유저 — 대시보드 직접 접근

| 단계 | 유저 행동 | 현재 결과 | 기대 결과 |
|------|----------|----------|----------|
| 1 | `/` 를 브라우저 북마크로 접근 | 로딩 중 빈 화면 → DisconnectedView | 로딩 스켈레톤 표시 → DisconnectedView (전환 명확) |
| 2 | "Gateway 연결하기" 클릭 | `/setup/1`로 이동 | 동일 |

### 시나리오 3: 설치 마법사 중간 이탈

| 단계 | 유저 행동 | 현재 결과 | 기대 결과 |
|------|----------|----------|----------|
| 1 | `/setup/1`에서 자동 감지 진행 중 | 감지 스피너 표시 | 동일 |
| 2 | 브라우저 뒤로가기 또는 사이드바 다른 메뉴 클릭 | 해당 페이지의 빈 상태 또는 에러 | 설치 미완료 안내 + "마법사로 돌아가기" 버튼 |
| 3 | 다시 마법사로 돌아오기 | `/setup/1`부터 다시 시작 | 이전 단계를 기억하고 해당 단계에서 재개 |

### 시나리오 4: Gateway 연결 후 정상 사용

| 단계 | 유저 행동 | 현재 결과 | 기대 결과 |
|------|----------|----------|----------|
| 1 | 설치 마법사 완료 → 대시보드 이동 | 대시보드 정상 표시 | 동일 |
| 2 | 사이드바 "설정" 클릭 | 설정 카드 표시 (클릭 가능) | 동일 (연결 상태이므로 빈 상태 안내 불필요) |

## 범위

### 포함

- `pages/+guard.ts`: Gateway 연결 상태 기반 리다이렉트 로직 추가
- `pages/index/+Page.tsx`: 로딩 상태의 스켈레톤 UI 추가
- 미연결 상태에서 접근 가능한 모든 페이지의 일관된 빈 상태 처리
- 재사용 가능한 "SetupRequired" 컴포넌트 제작

### 제외

- 설치 마법사 자체의 UX 개선 (이미 동작하는 기능)
- 에이전트 오피스의 데모/프리뷰 (별도 스프린트)
- 페이지 로딩 속도 최적화 (별도 스프린트)
- 사이드바 메뉴 구조 개편 (별도 스프린트)

## 성공 기준

1. 미연결 상태에서 어떤 페이지에 접근하든 3초 이내에 "설치 마법사로 이동" 행동이 가능하다
2. 미연결 상태에서 빈 화면(콘텐츠 없이 레이아웃만 보이는 상태)이 0.5초 이상 지속되지 않는다
3. `npm run build`가 에러 없이 통과한다
